HTML Lesson 1: Introduction to Web Development

What is the Web?

The World Wide Web (often simply called â€œthe webâ€) is a vast system of connected documents and multimedia content, all accessed via the Internet. When you open a browser like Chrome, Safari, or Firefox and type in a web address, youâ€™re using the web. It allows users to visit websites, access information, watch videos, shop, play games, and much moreâ€”all through web pages displayed on screen.

The web operates on a fundamental idea: request and response. When you visit a web page, your browser sends a request to a server, which responds with the data needed to display that page.

How the Web Works

At the heart of the web is a model called client-server architecture. Here's how it works: When you type a website address (also called a URL) into your web browser, your browser becomes the client. It sends a message to a web server, which is a computer that hosts the website. This message asks the server to send back the contents of the pageâ€”usually written in HTML, CSS, and JavaScript.

The server receives the request, finds the appropriate files, and sends them back to the browser. Your browser then reads the code and renders the page on your screen.

This process happens in seconds, often in less than a second, and it's repeated every time you visit a new page or interact with a website.

Understanding URLs

A URL stands for Uniform Resource Locator, and it's the address used to locate a resource on the web.

Consider this example URL:

https://www.example.com/about

Let's break it down:

  • https:// is the protocol, which tells the browser how to communicate with the server. Most modern sites use HTTPS (the secure version of HTTP).
  • www.example.com is the domain name, which maps to a specific server on the internet.
  • /about is the path, which points to a particular file or resource on that server.

Together, these parts tell your browser exactly where to go and what to request.

What is HTTP?

HTTP stands for HyperText Transfer Protocol. It is the foundation of communication on the web. When your browser requests a page, it uses HTTP to ask the server for the files it needs. The server, in turn, uses HTTP to send those files back.

HTTP is a stateless protocol, which means each request is independent. Every time your browser asks for something, the server responds without remembering previous requests (unless features like cookies or sessions are used).

Common HTTP Methods

HTTP defines several types of requests, known as methods. These tell the server what the browser wants to do.

  • GET: Used to retrieve data from the server. When you visit a website, your browser uses a GET request.
  • POST: Used to send data to the server, often when submitting a form (like logging in or signing up).
  • PUT: Used to update an existing resource on the server.
  • DELETE: Used to remove a resource.
  • HEAD: Similar to GET, but it only asks for the headers (metadata), not the actual content.

Understanding these methods is key to understanding how web applications interact with servers.

The Core Technologies of the Web: HTML, CSS, and JavaScript

Modern web pages are built using three primary technologies: HTML, CSS, and JavaScript. Each plays a distinct role.

HTML - The Structure

HTML stands for HyperText Markup Language, and it's the backbone of any web page. It defines the structure and contentâ€”such as headings, paragraphs, images, lists, and links.

For example:


    <h1>Welcome to My Site</h1>
    <p>This is my homepage.</p>

In this code, <h1>creates a large heading, and <p> defines a paragraph. Every visible element on a web page starts with HTML.

CSS - The Style

CSS stands for Cascading Style Sheets. It's responsible for how a web page looksâ€”the layout, fonts, colors, spacing, and overall design. Without CSS, web pages would look very plain and difficult to use.

For example:


h1 {
  color: navy;
  text-align: center;
}

This style makes the heading text navy blue and centers it on the page.

CSS allows web developers to create beautiful, responsive designs that work across devices.

JavaScript - The Behavior

JavaScript is a programming language used to add interactivity and dynamic behavior to web pages. It can respond to user actions like clicks, form submissions, and scrolling.

For example:


document.querySelector("h1").addEventListener("click", function() {
  alert("You clicked the heading!");
});

This script displays a pop-up alert when the user clicks on the heading. JavaScript powers many modern web featuresâ€”like image sliders, form validation, and real-time updates.

How HTML, CSS, and JavaScript Work Together

These three technologies work together seamlessly to create interactive, visually appealing web experiences:

  • HTML provides the content.
  • CSS styles and formats the content.
  • JavaScript
    • makes the content dynamic and interactive.

When your browser receives a webpage, it first processes the HTML to create the structure. Then, it applies the CSS to style it. Finally, it runs JavaScript to add any interactive features.

Recap and Key Concepts

To wrap up, here are the essential ideas from this lesson:

  • The web operates on a client-server model, where your browser (client) requests resources from a server.
  • HTTP is the protocol used to request and deliver resources.
  • A URL identifies the location of a resource on the web.
  • Web pages are built using HTML (structure), CSS (style), and JavaScript (interactivity).

Understanding these basics gives you a strong foundation for learning more advanced web development topics like frameworks, APIs, databases, and full-stack applications.
